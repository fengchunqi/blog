    NPM中--save、--save-dev、-S、-D的区别

    vuePress-theme-reco fengchunqi    2020 - 2021

    NPM中--save、--save-dev、-S、-D的区别

    fengchunqi 2021-4-29 ArticlesVueNPM

    tips

    参考文章：

    • npm install --save 、--save-dev 、-D、-S 的区别与NODE_ENV的配置 (opens new window)

    # 结论

    • --save 等价于 -S
    • --save-dev 等价于 -D

    就如同 npm install 等价于 npm i 一样，缩写写法。

    各种安装命令

    npm i xxxxx

npm i xxxxx -g

npm i xxxxx --save
npm i xxxxx -S

npm i xxxxx --save-dev
npm i xxxxx -D

    说明：

    1. npm i xxxxx 会把模块安装到 node_modules 目录下，但是不会写入模块信息到 package.json 中，所以在项目安装依赖时使用 npm i 不会安装到它
    2. npm i xxxxx -g 会把模块安装到全局，地址是 npm config get prefix 的地址，也不会写入模块信息到 package.json 中，在项目安装依赖时使用 npm i 也不会安装到它
    3. npm i xxxxx --save 和 npm i xxxxx -S 等价，都是把模块安装到 package.json 中的 dependencies 下，作为项目生产环境的依赖
    4. npm i xxxxx --save-dev 和 npm i xxxxx -D 等价，都是把模块安装到 package.json 中的 devDependencies 下，作为项目开发环境的依赖

    tips

    使用 npm i 安装 package.json 里的依赖时，dependencies 和 devDependencies 两部分的包都会pull下来，所以在一般情况下装哪里只是个规范问题，装错了项目依然可以运行。

    npm install --production (npm i --prod) 或者注明 NODE_ENV=production 时，只会下载 dependencies 中的模块。

    注意

    如果是发布包到npm仓库，别人下载使用，此时依赖装在哪里有很大的影响。

    比如 eslint, node-sass 这些只在开发环境使用的，应该装在 devDependencies 下，像 vue, vuex 这些在发布后依然需要使用，使用者在安装你所发布的包时，同时会安装 dependencies 下的依赖，不会去安装 devDependencies，如果依赖包信息位置错误将会导致项目无法正常使用。
