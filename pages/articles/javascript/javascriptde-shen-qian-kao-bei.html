    JavaScript的深浅拷贝

    fengchunqi 2021/6/17 JavaScript

    # 深浅拷贝的区别

    假设B复制了A，当修改B时，看A是否会发生变化，如果A也跟着变了，说明这是浅拷贝，如果A没变，那就是深拷贝。

    在JavaScript中深拷贝只针对引用类型的值（如array，object），而基本类型的值不存在深浅拷贝一说（如number，string，boolean），因此这里只考虑引用类型的数据：object和array。

    # 浅拷贝的实现

    # 直接赋值

    const arr1 = [1, 2, 3];
const arr2 = arr1;
console.log(arr2); // [1, 2, 3]
arr2[1] = 22;
console.log(arr1); // [1, 22, 3]
console.log(arr2); // [1, 22, 3]

    # 深拷贝的实现

    # 只有基本类型的数组

    • 创建新数组，循环往里添加
    • slice: const arr2 = arr1.slice(0);
    • concat: const arr2 = arr1.concat();
    • Array.from(): const arr2 = Array.from(arr1);
    • 扩展运算符: const arr2 = [...arr1];

    # 只有基本类型的对象

    • 创建新对象，循环往里添加
    • Object.assign(): const obj2 = Object.assign({}, obj1)
    • 扩展运算符: const obj2 = { ...obj1 };

    # 数组或对象里面还有引用数据

    • JSON.parse(JSON.stringify())

      const obj1 = { name: 'obj1', child: { name: 'child', } };
const obj2 = JSON.parse(JSON.stringify(obj1));

      2

      WARNING

      JSON.parse(JSON.stringify())实现深拷贝对象（object）有弊端：

      • 时间对象Date，格式化之后变成字符串。
      • RegExp，Error对象，格式化后变成空对象{}。
      • 函数，undefined，Symbol格式化后键值对将丢失。
      • NaN，Infinity，-Infinity，格式化后变成null。
      • JSON.stringify()只能序列化对象的可枚举的自有属性，如果对象中的属性值是由构造函数生成的，格式化会丢弃对象的constructor。
      • 如果对象中存在循环引用的情况也无法正确实现深拷贝。

    • 递归

    function getDataType(value) {
  return (Object.prototype.toString.call(value).slice(8, -1) || '').toLowerCase();
}
/**
 * @description 深拷贝
 * @author fengchunqi
 * @date 2021-06-17 14:35:13
 * @param {*} target
 * @param {Map} [cache=new Map()]
 * @returns {*}
 */
function deepClone(target, cache = new Map()) {
  if (cache.has(target)) {
    return cache.get(target);
  }
  const type = getDataType(target);
  let result;
  if (type === 'array') {
    result = [];
    // 如果出现重复引用同一个值可以直接取缓存
    cache.set(target, result);
    target.forEach((el) => {
      result.push(deepClone(el, cache));
    });
  } else if (type === 'regexp') {
    result = new RegExp(target.source, target.flags);
    cache.set(target, result);
  } else if (type === 'date') {
    result = new Date(target);
    cache.set(target, result);
  } else if (type === 'object') {
    result = {};
    cache.set(target, result);
    // 如果需要拷贝Symbol键，需要合并普通key和Symbol的key
    // Object.getOwnPropertyNames(target).concat(Object.getOwnPropertySymbols(target))
    Object.getOwnPropertyNames(target).forEach((key) => {
      result[key] = deepClone(target[key], cache);
    });
  } else {
    result = target;
    cache.set(target, result);
  }
  return result;
}

const obj = {
  Number: 1,

    name: 'object',

  },

const copy = deepClone(obj);

    WARNING

    上述递归方法还是不够完善，如果是Map，Set或者DOM节点，或者里面还有引用类型的数据，还是没有深度拷贝，处理方式可以像数组对象那样，新开辟一个相同数据类型的空间，一个个往里面加。所以说深度克隆需要看真实的业务场景。
