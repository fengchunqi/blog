    Git分支与tag的同步

    fengchunqi 2021/5/20 Git

    TIPS

    参考文章：

    • git 如何同步本地tag与远程tag (opens new window)

    # 分支同步

    git分支操作

    # 查看本地分支
git branch

# 查看远程分支
git branch -r

# 查看所属分支
git branch -a

# 创建新分支
git branch [分支]

# 切换分支
git checkout [分支]

# 创建并切换到该分支
git checkout -b [分支]

# 合并分支（合并分支之前需要切换到master分支上）
git merge [分支]

# 删除本地分支
git branch -d [分支]
git branch -D [分支]
# git branch -d 会在删除前检查merge状态（其与上游分支或者与head）
# git branch -D 是 git branch --delete --force 的简写，它会直接删除。

# 删除远程仓库的分支
git push origin --delete [分支]

    # 远程仓库已经删除的分支还在

    远程仓库删除了分支，通过命令 git branch -r 查看还在

    使用命令 git remote show origin ，可以查看remote地址，远程分支，还有本地分支与之相对应关系等信息。

    可以看到有些分支是标记为 tracked，即该分支是被跟踪的，有些则标记为 stale(use 'git remote prune')，意为不新鲜的旧的，就是已经被删除了的分支，按照提示使用 git remote prune origin 即可删除

    # tag同步

    TIPS

    问题:

    在远程仓库删除了某个tag，在本地拉取，此时被删除的tag在本地还是存在，再次 push 的时候又会推到远程。

    也有那种在远程把某个tag删了，然后又打了一个名字一样的，导致后续推送就要出问题。

    tag常用git命令：

    # 删除所有本地tag
git tag -l | xargs git tag -d
# git tag -l ---->  列出所有本地tag
# git tag -d xxx ---->  删除制定tag
# xargs 前面一个指令（git tag -l）的输出，作为后面一个指令（git tag -d）的传参

# 从远程拉取所有信息
git fetch origin --prune

# 查询远程tags的命令如下：

git ls-remote --tags origin

# 列出所有tag
git tag

# 列出符合条件的tag（筛选作用）
git tag -l v1.*

# 创建轻量tag（无-m标注信息）
git tag [tag名]

# 创建含注解的tag
git tag -a [tag名]

# 推送所有本地tag到远程
git push origin --tags

# 推送指定本地tag到远程
git push origin [本地tag名]

# 删除本地指定tag
git tag -d [本地tag名]

# 删除远程指定tag
git push origin :refs/tags/[远程tag名]

# 拉取远程指定tag
git fetch origin [远程tag名]

# 显示指定tag详细信息
git show [tag名]

    解决办法：

    git tag -l | xargs git tag -d
git fetch origin --prune
